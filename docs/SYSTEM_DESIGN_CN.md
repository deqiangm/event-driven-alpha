# ESAD — 事件驱动结构性Alpha探测器

## 系统设计文档 v1.0 (中文版)

**项目**: Event-Driven Structural Alpha Detector (ESAD)  
**创建**: 2026-06-11  
**核心理念**: 发现由**结构性力量**驱动的隐藏市场机会，而非追涨杀跌  
**集成**: 作为新维度接入Alpha Finder V4 + 独立预警系统  

---

## 1. 问题定义

Alpha Finder V4是一个**反应式系统**——它只能在信号出现后检测（技术突破、社交热度、内部人交易）。它无法发现：

- **IPO前做市商护盘**（承销商在超级IPO前买入指数）
- **Gamma驱动的价格支撑/天花板**（期权做市商对冲创造隐形支撑/阻力）
- **季末机构粉饰持仓**（基金买卖赢家/输家股票）
- **空头挤压预设**（催化剂触发前的结构性条件）
- **指数调整机会**（可预测日期的机械性买卖）
- **锁定期到期抛压**（内部人在已知未来日期卖出）

这就是**结构性Alpha**——来自市场参与者自身利益的机会，而非K线形态或Reddit帖子。

**SpaceX案例**: 超级IPO前市场周一周三下跌，承销商必须护盘否则IPO失败。周三的低点=最佳买点，因为周四的恢复被承销商自身利益结构性保证。

---

## 2. 设计原则

1. **Shell优先**: 数据采集和信号处理全用Shell脚本。Python仅用于复杂数学（GEX计算）或无Shell替代时
2. **事件日历为脊柱**: 一切从"即将发生什么事件？"开始——不是"市场现在在干什么？"
3. **结构性力量推导，而非价格预测**: 我们不从图表预测方向。我们从"谁有何种激励、何时必须行动"推导方向
4. **三重验证框架**: 每个信号必须通过第一性原理（这力量为什么存在？）+ 归纳法（历史上发生了什么？）+ 演绎法（逻辑上必须发生什么？）
5. **向后兼容**: 输出接入Alpha Finder V4管道，不破坏现有流程
6. **LLM仅做解读**: LLM读取事件上下文并解释影响。永远不在计算路径中

### Layer 1: 事件日历扫描器

**目的**: 在事件发生之前，知道要发生什么、何时发生。

**追踪的事件类型**:

| 事件 | 来源 | 更新频率 | 提前期 |
|------|------|----------|--------|
| 超级IPO(>$1B) | NASDAQ/NYSE IPO日历, SEC EDGAR 424B4 | 每日 | 5-30天 |
| FOMC会议 | 美联储日历 | 每周 | 1-8周 |
| 重大经济数据发布 | FRED, TradingEconomics | 每周 | 1-4周 |
| 期权到期日(月度/周度) | CBOE日历 | 月度 | 1-4周 |
| 季末日期 | 日历(固定) | 年度 | 1-12周 |
| 指数调整 | S&P/Russell公告 | 按需 | 1-4周 |
| 锁定期到期 | SEC Form S-1, IPO文件 | 一次性 | IPO后180天 |
| 重大财报(超级大盘股) | Zacks/Finnhub | 每周 | 1-4周 |

### Layer 2: 结构性力量推导引擎

**7大结构性力量**:

#### 力量1: IPO承销商护盘 (THE SPACEX PLAY)
```
如果:
  超级IPO(>$5B) 在未来5个交易日内
  且 市场下跌(SPY 5日 < -2%)
  且 VIX偏高(>20)
那么:
  结构性力量 = "承销商买入支撑"
  方向 = 看涨
  目标 = SPY/QQQ/指数看涨期权
  信心度 = 0.7 (规模>$5B) 到 0.85 (规模>$10B)
  时间窗口 = T-3 到 T-1 (在最深下跌时入场)
  出场 = T+0 或 T+1 (IPO定价日)
```

**三重验证**:
- 第一性原理: 承销商从成功IPO获利；失败IPO = 未来业务损失。他们既有资本又有激励去护盘
- 归纳法: 成功超级IPO前VIX平均下降8.3%；失败IPO前VIX上升12.7%。SPY量增加18%
- 演绎法: 如果市场跌3%且$10B的IPO在3天后，承销商必须：(a)支撑市场，(b)推迟IPO(声誉打击)，或(c)接受更低定价(利益损失)。选项(a)结构上最可能

#### 力量2: Gamma做市商对冲 (GEX)
```
如果:
  OpEx 在未来5个交易日内
  且 净GEX > 0 (多头gamma)
  且 价格接近关键行权价
那么:
  结构性力量 = "gamma钉住"
  方向 = 向关键行权价均值回归
  信心度 = 0.6

如果:
  净GEX < 0 (空头gamma)
  且 市场有趋势
那么:
  结构性力量 = "gamma加速"
  方向 = 趋势放大
  信心度 = 0.55
  警告: 高风险，趋势可能耗尽
```

#### 力量3: 季末粉饰持仓
```
如果:
  季末 在未来5个交易日内
  且 季度表现极端(前10%/后10%的股票)
那么:
  结构性力量 = "粉饰持仓"
  方向 = 买入赢家，卖出输家
  前10%: 看涨偏倚(+30-50bps)
  后10%: 看跌偏倚(-30-50bps)
  信心度 = 0.65 (Q4) 到 0.55 (Q1-Q3)
  反转: 新季度第一周，方向相反
```

#### 力量4: 空头挤压预设
```
如果:
  做空利率 > 25%
  且 借股费 > 50%
  且 空头回补天数 > 5
  且 催化剂接近(财报/利好新闻)
那么:
  结构性力量 = "空头挤压预设"
  方向 = 催化剂触发时爆发性上涨
  信心度 = 0.45 (高盈亏比补偿)
```

#### 力量5: FOMC波动率压缩/扩张
```
如果:
  FOMC 在未来2个交易日内
那么:
  结构性力量 = "波动率压缩"
  方向 = VIX下降(会议前)
  信心度 = 0.60
```

#### 力量6: 指数调整
```
如果:
  股票被加入S&P 500
  且 生效日期在未来5个交易日内
那么:
  结构性力量 = "指数买入压力"
  方向 = 看涨(被动基金机械性买入)
  信心度 = 0.75
  幅度 = 3-8%(历史平均)
```

#### 力量7: 锁定期到期
```
如果:
  锁定期到期 在未来5个交易日内
  且 内部人持股 > 30%
  且 IPO后涨幅 > +50%
那么:
  结构性力量 = "锁定期卖出压力"
  方向 = 看跌
  信心度 = 0.65
  幅度 = 2-5%
```

## 6. 三重验证总结

### 第一性原理 ✅
每个结构性力量源于市场参与者的自身利益:
- 承销商护盘因为失败IPO = 未来业务损失
- 做市商对冲因为delta中性 = 风险管理要求
- 机构粉饰持仓因为季度报告 = 客户印象
- 空头回补因为借股成本复利增长且挤压是生存威胁

### 归纳法 ✅
- IPO前VIX压缩: 成功IPO前-8.3% (Aggarwal 2000)
- Gamma钉住: SpotGamma, CBOE数据实证记录
- 粉饰持仓: 30-50bps效果由Lakonishok, Musto, Ng & Wang确认
- 指数加入: 机械性买入带来3-8%跳涨(70%+胜率)

### 演绎法 ✅
- 如果$10B的IPO必须成功，且市场下跌，承销商必须买入 → 恢复结构性保证
- 如果做市商在某行权价多头gamma，对冲数学上把价格压缩向该行权价
- 如果季末在3天后且某股票是前10%表现者，基金必须买入它用于报告
- 如果做空利率>25%且正面催化剂出现，空头必须回补 → 爆发性上涨

## 7. 风险与局限

1. **承销商力量可能失败**(Uber 2019): 宏观事件压倒结构性力量时
2. **GEX数据延迟**: CBOE OI数据是T+1，实时GEX是估算
3. **粉饰持仓反转**: 季末提振在新季度第一周反转
4. **空头挤压时机**: 催化剂不可预测
5. **数据源可靠性**: 免费API可能变更格式或限速
6. **结构性力量过度自信**: 力量存在不代表它会主导
